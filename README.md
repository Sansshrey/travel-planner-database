# Travel Planner Database (SQL Project)

## Project Overview
The Travel Planner Database is a SQL-based project designed to manage
travel destinations, trips, and budget information. The project enables
efficient cost analysis and travel planning using relational database
concepts and analytical SQL queries.

---

## Database Design
The database follows normalization principles and consists of:

- destinations: Stores city, country, and average daily cost
- trips: Stores trip duration, destination mapping, and number of travelers
- budgets: Stores transport, accommodation, and food expenses

Foreign key relationships ensure data integrity and efficient joins.

---

## SQL Concepts & Skills Used
- Relational Database Design
- Normalization
- Primary and Foreign Keys
- INNER JOIN operations
- Aggregate Functions (SUM, AVG)
- GROUP BY and ORDER BY
- Filtering using WHERE clause
- Cost Analysis Queries

---

## Key Analytical Queries
- Calculate total cost per trip
- Identify the most expensive trip
- Find budget-friendly trips
- Analyze average trip cost by country
- Sort trips based on duration
